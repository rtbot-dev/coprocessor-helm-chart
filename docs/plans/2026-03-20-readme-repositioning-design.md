# README Repositioning Design

## Goal

Make the public `coprocessor-helm-chart` README more compelling for new users, normalize naming to `RtBot`, and lead with the value proposition plus the fastest install path.

## Approach

- normalize public documentation wording from `RTBot` to `RtBot`
- restructure the README so the top answers:
  - what it does
  - why you want it
  - how to install it quickly
  - how to verify it basically worked
- keep deeper configuration and operational detail lower in the document
- include a compact table of contents if it helps scanability

## Success Criteria

- the README feels like a public product landing page, not only a technical reference
- new users can find the install path in seconds
- naming is consistent across the chart documentation
